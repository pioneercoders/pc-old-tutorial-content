						<h3 class="subheading">Application Context :</h3>
						
					<p>The Application Context is spring's more advanced container. Similar to BeanFactory it can load bean definitions, wire beans together and dispense beans upon request. Additionally it adds more enterprise-specific functionality such as the ability to resolve textual messages from a properties file and the ability to publish application events to interested event listeners. This container is defined by the org.springframework.context.ApplicationContext interface.</p>
					<p>The ApplicationContext includes all functionality of the BeanFactory, it is generally recommended over the BeanFactory. BeanFactory can still be used for light weight applications like mobile devices or applet based applications.</p>
					<p>The most commonly used ApplicationContext implementations are:</p>
					<p class="padding-left"><h4>FileSystemXmlApplicationContext:</h4> This container loads the definitions of the beans from an XML file. Here you need to provide the full path of the XML bean configuration file to the constructor.</p>
					<p class="padding-left"><h4>ClassPathXmlApplicationContext This container loads the definitions of the beans from an XML file. Here you do not need to provide the full path of the XML file but you need to set CLASSPATH properly because this container will look bean configuration XML file in CLASSPATH.</p>
					<p class="padding-left"><h4>WebXmlApplicationContext:</h4> This container loads the XML file with definitions of all beans from within a web application.</p>
					<p>We already have seen an example on ClassPathXmlApplicationContext container in Spring Hello World Example, and we will talk more about XmlWebApplicationContext in a separate chapter when we will discuss web based Spring applications. So let see one example on FileSystemXmlApplicationContext.</p>
					<p><b>Exapmle</b></p>

					<p>There are following two important points to note about the main program:</p>
					<p>First step is to create factory object where we used framework API <b>FileSystemXmlApplicationContext</b> to create the factory bean after loading the bean configuration file from the given path. The <b>FileSystemXmlApplicationContext()</b> API takes care of creating and initializing all the objects ie. beans mentioned in the XML bean configuration file.</p>
					<p>Second step is used to get required bean using <b>getBean() </b>method of the created context. This method uses bean ID to return a generic object which finally can be casted to actual object. Once you have object, you can use this object to call any class method.</p>
