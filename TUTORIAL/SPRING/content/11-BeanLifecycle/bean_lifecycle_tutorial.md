						<h3 class="subheading">Bean Lifecycle :</h3>
						
					<p>When bean is initialized it might require to perform some activity before it can come into use able state(State in which application can use it) and when bean is getting destroyed there might be some cleanup activity required for given bean. These activities are known as bean Lifecycle.</p>
					<p>Standard bean lifecycle interfaces & there standard order of execution are given below.</p>
					<p class="padding-left">1.IoC container will look for the configuration metadata of given Bean.</p>
					<p class="padding-left">2.Once find, container will create the instance of Bean(Using reflection API).</p>
					<p class="padding-left">3. After instance creation dependency will be injected(DI).</p>
					<p>If Bean Class  implements any of the below interface then corresponding method will be invoked in below order</p>
					<p class="padding-left">4.setBeanName method of BeanNameAware class. It sets the name of the bean in the bean factory that created this bean.</p>
					<p class="padding-left">5.setBeanClassLoader method of BeanClassLoaderAware class. Callback that supplies the bean to a bean instance.</p>
					<p class="padding-left">6.setBeanFactory  method of BeanFactoryAware class. Callback that supplies the owning factory to a bean instance.</p>
					<p>Below method execution will be applicable when running in an application context</p>
					
					<p class="padding-left">7.setResourceLoader  method of ResourceLoaderAware class. It set the ResourceLoader that this object runs in.</p>
					<p class="padding-left">8.setApplicationEventPublisher  method of ApplicationEventPublisherAware class. Set the ApplicationEventPublisher that this object runs in.</p>
					<p class="padding-left">9.setMessageSource method of MessageSourceAware class. Set the MessageSource that this object runs in.</p>
					<p class="padding-left">10.setApplicationContext method of ApplicationContextAware class. Set the ApplicationContext that this object runs in.</p>
					<p class="padding-left">11.setServletContext method of ServletContextAware. Set the ServletContext that this object runs in.</p>
					<p class="padding-left">12.postProcessBeforeInitialization method of BeanPostProcessor. Apply this BeanPostProcessor to the given new bean instance before any bean initialization callbacks.</p>
					<p class="padding-left">13.afterPropertiesSet method of InitializingBean. Invoked by a BeanFactory after it has set all bean properties supplied</p>
					<p>In case Bean class has custom init method defined(via init-method attribute</p>
					<p>Custom init method will be invoked.</p>
					<p class="padding-left">14.postProcessAfterInitialization methods of BeanPostProcessors. Apply this BeanPostProcessor to the given new bean instance after any bean initialization callbacks.</p>
					<p>When Bean Factory is getting shut down following lifecycle methods will be executed.</p>
					<p><b>Custome destroy</b> method will be executed if there is any defined via destroy-method attributes.</p>
