				<h3 class="subheading">Dependency Injection :</h3>
				<p>The basic concept of the dependency injection (also known as Inversion of Control pattern) is that you do not create your objects but
				describe how they should be created. You don't directly connect your components and services together in code but describe which services 
				are needed by which components in a configuration file. A container (in the case of the Spring framework, the IOC container) is then 
				responsible for hooking it all up.i.e., Applying IoC, objects are given their dependencies at creation time by some external entity that
				coordinates each object in the system. That is, dependencies are injected into objects. So, IoC means an inversion of responsibility with
				regard to how an object obtains references to collaborating objects. </p>
				<img src="image.jpg" alt="image">
				<p><b>Example :</b></p>
				<p> Lets we have two classes-Car and Engine.Car has a object of Engine.</p>
				<h3 class="subheading">Normal Way :</h4>
				<p>There are many ways to instantiate a object. A simple and common way is with new operator.There are many ways to instantiate a object. A simple and common way is with new operator.</p>
				<p>so here Car class contain object of Engine and we have it instantiated using new operator.</p>
		<!-- 		<img src="image.jpg" alt="image" /> -->
				<h3 class="subheading">Without DI</h3>
				<p class="padding-left"><b>With help of Dependency Injection:</b></p>
				<p class="padding-left">Now we outsource instantiation and supply job of instantiating to third party.Car needs object of Engine 
					to operate but it outsources that job to some third party. The designated third party, decides the moment of instantiation and the type
					to use to create the instance. The dependency between class Car and class Engine is injected by a third party. Whole of this agreement 
					involves some configuration information too. This whole process is called dependency injection.
				</p>
			<!-- 	<img src="image.jpg" alt="image" /> -->
				<h3 class="subheading">With DI</h3>
				<p class="padding-left"><b>BENIFITS OF DEPENDENCY INJECTION IN SPRING:</b></p>
				<p class="padding-left">1.Ensures configuration and uses of services are separate.</p>
				<p class="padding-left">2.Can switch implementations by just changing configuration.</p>
				<p class="padding-left">3.Enhances Testability as mock dependencies can be injected.</p>
				<p class="padding-left">4.Dependencies can be easily identified.</p>
				<p class="padding-left">5.No need to read code to see what dependencies your code talks to.</p>
				<p class="padding-left"><b>TYPES OF DEPENDENCY INJECTION:</b></p>
				<h4 class="padding-left">Setter Injection :</h4>
				<p class="padding-left">Setter-based DI is realized by calling setter methods on your beans after invoking a no-argument constructor or no-argument static factory method to instantiate your bean.</p>
				<h4 class="padding-left">Constructor Injection :</h4>
				<p class="padding-left">Constructor-based DI is realized by invoking a constructor with a number of arguments, each representing a collaborator.</p>
				<h4 class="padding-left">Interface Injection :</h4>
				<p class="padding-left">In interface-based dependency injection, we will have an interface and on implementing it we will get the instance injected.</p>
