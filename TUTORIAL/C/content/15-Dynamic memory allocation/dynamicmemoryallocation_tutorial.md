	<h3> DYNAMIC MEMORY ALLOCATION</h3>
	<p>	The exact size of array is unknown until the compile time, i.e., time when a compiler compiles code written in a programming language into a executable form. The size of array you have declared initially can be sometimes insufficient and sometimes more than required.</p>
	
	<p>Dynamic memory allocation allows a program to obtain more memory space, while running or to release space when no space is required.</p>
	<p>Although, C language inherently does not have any technique to allocated memory dynamically, there are 4 library functions under "stdlib.h" for dynamic memory allocation.</p>

		<h4>malloc( )</h4>
		<p>The name malloc stands for "memory allocation". The function malloc() reserves a block of memory of specified size and return a pointer of type void which can be casted into pointer of any form.</p>		
		@CODE_START@@C@ptr = (cast-type*) malloc(byte-size);@CODE_END@	
		<p>Here, ptr is pointer of cast-type. The malloc() function returns a pointer to an area of memory with size of byte size. If the space is insufficient, allocation fails and returns NULL pointer.</p>

		@CODE_START@@C@ptr = (int *) malloc(100*sizeof(int));@CODE_END@	
		<p>This statement will allocate either 200 or 400 according to size of int 2 or 4 bytes respectively and the pointer points to the address of first byte of memory.</p>
		
		<h4>calloc( )</h4>
		
		<p>The name calloc stands for "contiguous allocation". The only difference between malloc() and calloc() is that, malloc() allocates single block of memory whereas calloc() allocates multiple blocks of memory each of same size and sets all bytes to zero.</p>
		@CODE_START@@C@ptr = (cast-type*) calloc(n,element_size);@CODE_END@	
		<p>This statement allocates contiguous space in memory for an array of 25 elements each of size of float, i.e, 4 bytes.</p>

	<h4>free()</h4>
		
	<p>Dynamically allocated memory with either calloc( ) or malloc( ) does not get return on its own. The programmer must use free( ) explicitly to release space.</p>
	@CODE_START@@C@ free(ptr);@CODE_END@		
	<p>This statement causes the space in memory pointer by ptr to be deallocated.</p>
		
	<h4>relloac()</h4>
	<p>If the previously allocated memory is insufficient or more than sufficient. Then, you can change memory size previously allocated using realloc().</p>
	@CODE_START@@C@ ptr = realloc(ptr,newsize);@CODE_END@		
	<p>Here, ptr is reallocated with size of newsize.</p>
